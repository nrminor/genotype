    findPatternBatch

    • findPatternBatch(
          sequences: Buffer,
          offsets: Uint32Array,
          pattern: Buffer,
          maxEdits: number,
          caseInsensitive: boolean,
      ): PatternSearchResult

      Find all pattern matches with positions and edit distances in a batch of sequences.

      Uses the Iupac profile (forward-only) with traceback enabled, so IUPAC degenerate bases are handled correctly and exact match start positions are computed. The caller handles orientation by making separate calls with the original and reverse-complement patterns.

      Parameters

      • sequences: Buffer

        Concatenated sequence bytes

      • offsets: Uint32Array

        N+1 offset array into the sequences buffer

      • pattern: Buffer

        Pattern to search for

      • maxEdits: number

        Maximum edit distance

      • caseInsensitive: boolean

        Whether to ignore case (Iupac profile is inherently case-insensitive)

      Returns PatternSearchResult

      CSR-style result with match positions and costs

          trimStart: boolean,

      Find trim positions for each sequence in a batch using a sliding window average quality threshold.

      Returns a flat array of length numSequences * 2, where result[i*2] is the start position and result[i*2+1] is the end position (exclusive). A (0, 0) pair means trimming consumed the entire sequence.

      Parameters

      • quality: Buffer

        Concatenated quality bytes

      • offsets: Uint32Array

        N+1 offset array into the quality buffer

      • asciiOffset: number

        ASCII offset to subtract (33 or 64)

      • threshold: number

        Minimum average quality score for a window to pass

      • windowSize: number

        Number of bases in the sliding window

      • trimStart: boolean

        Whether to trim from the 5' end

      • trimEnd: boolean

        Whether to trim from the 3' end

      Returns number[]

      Flat array of start/end pairs, two per sequence

      ): TransformResult

      Remap quality bytes into fewer bins using SIMD compare-and-select.

      boundaries and representatives are raw ASCII byte values (pre-offset-adjusted by the caller). The kernel does pure byte comparisons with no encoding awareness.

      Parameters

      • quality: Buffer

        Concatenated quality bytes

      • offsets: Uint32Array

        N+1 offset array into the quality buffer

      • boundaries: Buffer

        ASCII byte thresholds between bins (length 1-4)

      • representatives: Buffer

        ASCII byte values for each bin (length = boundaries.length + 1)

      Returns TransformResult

      Remapped bytes and identical offsets (length-preserving)
